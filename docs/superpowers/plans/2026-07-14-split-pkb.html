Goal: Shrink lib/pkb.sh (1133) to a ~270-line orchestrator by relocating whole functions verbatim into 3 focused modules (pkb-cache.sh, pkb-query.sh, pkb-prompts.sh), with zero behavior change.

Architecture: Behavior-preserving relocation. A validated Python partition helper slices pkb.sh into per-function blocks (leading comments + body, brace-agnostic — boundary is the next function's start, NOT a ^}$) and moves a named subset into a new module, rewriting pkb.sh without them. Each task moves one module's functions AND wires its source line so the repo stays working after every task. Correctness is gated by a line-multiset diff (safe against the heredoc-} trap), per-module function-set checks, shellcheck, and the full suite.

Tech Stack: Bash, Python 3 (partition helper only), shellcheck, ./test.sh.

Global Constraints

  • Behavior-preserving only: functions move byte-for-byte. No logic/signature/prompt change. Guarded by tests/test_pkb_*.sh + shellcheck -S error.
  • DO NOT use the awk ^}$ heuristic. pkb.sh has 27 functions but 28 column-0 } — one lives inside a _pkb_generate_* PROMPT heredoc (the #12 Task-5 trap). Extraction MUST use the partition helper below (next-function-start boundary) and be verified by a line-multiset diff.
  • Repo stays working after each task: every task moves functions AND adds the new module's source line to bin/mra.sh (and to the 3 tests that source pkb.sh). Sourcing order is irrelevant (all lib/*.sh are function-definition files sourced before main(); calls resolve at call-time).
  • Function accounting: 10 (cache) + 4 (query) + 10 (prompts) + 3 stay = 27. After all tasks, pkb.sh defines exactly pkb_generate, pkb_incremental_update, pkb_capture_decisions.
  • ./test.sh green with unchanged counts throughout; shellcheck -S error clean on every new/modified file.

Shared mechanism — the partition helper

Save this to a scratch path (NOT committed), e.g. /private/tmp/.../pkb_move.py, and reuse it in every task. It is validated to reconstruct pkb.sh losslessly (header + 27 blocks == original).

#!/usr/bin/env python3
# pkb_move.py <src> <new> <header> <fn1> [fn2 ...]
# Move each named function's block (leading comments + body, brace-agnostic; block
# boundary = next function's start) from <src> into <new>; rewrite <src> without them.
import re, sys
src, new, header = sys.argv[1], sys.argv[2], sys.argv[3]
move = set(sys.argv[4:])
lines = open(src).read().splitlines(keepends=True)
fre = re.compile(r'^([A-Za-z_][A-Za-z0-9_]*)\(\) \{')
starts = [i for i, l in enumerate(lines) if fre.match(l)]
names  = [fre.match(lines[i]).group(1) for i in starts]
def bstart(k):
    i = starts[k]
    while i > 0 and lines[i-1].lstrip().startswith('#'):
        i -= 1
    return i
bs = [bstart(k) for k in range(len(starts))]
head = lines[:bs[0]]
blocks = [(names[k], lines[bs[k]:(bs[k+1] if k+1 < len(starts) else len(lines))])
          for k in range(len(starts))]
missing = move - set(names)
if missing:
    sys.exit("unknown functions: %s" % sorted(missing))
kept  = head + [l for n, b in blocks if n not in move for l in b]
moved = [l for n, b in blocks if n in move for l in b]
open(src, 'w').write(''.join(kept))
with open(new, 'w') as f:
    f.write("#!/usr/bin/env bash\n# %s\n\n" % header)
    f.write(''.join(moved))

diff /tmp/pkb_before.txt /tmp/pkb_after.txt && echo "LINE-MULTISET OK"
# 2) each moved fn present once in new module, gone from pkb.sh
for fn in <fns>; do
  [ "$(grep -c "^${fn}() {" lib/<newmodule>.sh)" = 1 ] || echo "MISSING: $fn"
  [ "$(grep -c "^${fn}() {" lib/pkb.sh)" = 0 ] || echo "STILL IN PKB: $fn"
done
# 3) lint + syntax + tests

Note: for step 1 across successive tasks, <TASK_BASE> is this task's starting commit (the previous task's HEAD), so the diff compares only THIS task's move. After the module is written, normalize its trailing newline to match siblings: printf '%s\n' "$(cat lib/<newmodule>.sh)" > /tmp/x && mv /tmp/x lib/<newmodule>.sh.
